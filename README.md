## 🛠 Технологии

- **Node.js** - среда выполнения JavaScript
- **Express.js** - веб-фреймворк
- **SQLite3** - легковесная база данных
- **JWT (jsonwebtoken)** - аутентификация и авторизация
- **bcrypt** - хеширование паролей
- **express-validator** - валидация входных данных
- **cors** - обработка CORS-запросов
- **dotenv** - управление переменными окружения
- **swagger-ui-express** - документация API
- **yaml** - парсинг YAML-файлов
- **concurrently** - одновременный запуск нескольких скриптов
- **nodemon** - автоматическая перезагрузка сервера при разработке

## 🚀 Установка и запуск

### Предварительные требования

- Node.js (версия 14 или выше)
- npm (обычно устанавливается вместе с Node.js)

### Шаги для запуска

1. **Клонируйте репозиторий**
```bash
git clone <repository-url>
cd analogapi
```

2. **Установите зависимости**
```bash
npm install
```

3. **Настройте переменные окружения**
   
   Создайте файл `.env` в корне проекта:
```env
PORT=7757
DATABASE_NAME=Film.db
JWT_ACCESS_SECRET=your-secret-access-key
JWT_REFRESH_SECRET=your-secret-refresh-key
```

4. **Подготовьте базу данных**
   
   Убедитесь, что у вас есть файл базы данных SQLite с необходимыми таблицами:
   - `Film` - таблица фильмов
   - `Actors` - таблица актеров
   - `genres` - таблица жанров
   - `genresfilm` - связь фильмов с жанрами
   - `Cast` - связь фильмов с актерами
   - `posters` - постеры фильмов
   - `Description` - описания фильмов
   - `users` - пользователи
   - `refresh_tokens` - refresh токены
   - `favorites` - избранные фильмы

5. **Запустите сервер**

   Режим разработки (с автоматической перезагрузкой):
```bash
npm run dev
```

   Обычный запуск:
```bash
npm run server
```

6. **Проверьте работу API**

   - API доступно по адресу: `http://127.0.0.1:7757`
   - Swagger документация: `http://127.0.0.1:7757/api-docs`
   - Статические файлы: `http://127.0.0.1:7757/images/...`

## 🔧 Переменные окружения

| Переменная | Описание | Пример |
|------------|----------|---------|
| `PORT` | Порт для сервера | `7757` |
| `DATABASE_NAME` | Имя файла базы данных SQLite | `movies.db` |
| `JWT_ACCESS_SECRET` | Секретный ключ для access токенов | `your-secret-access-key` |
| `JWT_REFRESH_SECRET` | Секретный ключ для refresh токенов | `your-secret-refresh-key` |

## 📊 Структура базы данных

Основные таблицы:
- `Film` - фильмы (id, title, year, rating)
- `Actors` - актеры (id, name, surname, age, Biography, image)
- `genres` - жанры (id, name)
- `users` - пользователи (id, email, password, username, role)
- `refresh_tokens` - refresh токены (user_id, token, expires_at)
- `favorites` - избранное (user_id, film_id)

## 📡 API Эндпоинты

Базовый URL: `http://127.0.0.1:7757`

### 🔐 Аутентификация

| Метод | Эндпоинт | Описание | Требует аутентификацию |
|-------|----------|----------|----------------------|
| POST | `/register` | Регистрация нового пользователя | ❌ |
| POST | `/login` | Вход в систему | ❌ |
| POST | `/refresh-token` | Обновление access токена | ❌ |
| POST | `/logout` | Выход из системы | ❌ |

### 🎬 Фильмы

| Метод | Эндпоинт | Описание | Требует аутентификацию |
|-------|----------|----------|----------------------|
| GET | `/films` | Получить все фильмы | ❌ |
| GET | `/top25` | Получить топ-25 фильмов по рейтингу | ❌ |
| GET | `/films/{id}` | Получить фильм по ID с жанрами и актерами | ❌ |

### 🎭 Актеры

| Метод | Эндпоинт | Описание | Требует аутентификацию |
|-------|----------|----------|----------------------|

| GET | `/actors/{id}` | Получить актера по ID со списком фильмов | ❌ |

| Метод | Эндпоинт | Описание | Требует аутентификацию |
|-------|----------|----------|----------------------|
| GET | `/favorites` | Получить избранные фильмы пользователя | ✅ |
| POST | `/favorites` | Добавить фильм в избранное | ✅ |
| DELETE | `/favorites` | Удалить фильм из избранного | ✅ |

## 🔑 Аутентификация

API использует **JWT (JSON Web Tokens)** для аутентификации:

### Типы токенов:
- **Access Token** - короткоживущий токен (15 минут)
- **Refresh Token** - долгоживущий токен (7 дней)

### Процесс аутентификации:

1. **Регистрация/Вход**
   - Пользователь отправляет email, пароль (и имя при регистрации)
   - Сервер возвращает access token, refresh token и данные пользователя

2. **Использование API**
   - Добавьте access token в заголовок `Authorization: Bearer <access-token>`
   - Используйте для защищенных эндпоинтов

3. **Обновление токена**
   - Когда access token истекает, отправьте refresh token на `/refresh-token`
   - Получите новый access token

4. **Выход**
   - Отправьте refresh token на `/logout` для его удаления из базы данных

## 📝 Примеры запросов

### Регистрация
```bash
curl -X POST http://127.0.0.1:7757/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123",
    "username": "JohnDoe"
  }'
```

### Вход
```bash
curl -X POST http://127.0.0.1:7757/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123"
  }'
```

### Получение избранного (требует аутентификацию)
```bash
curl -X GET http://127.0.0.1:7757/favorites \
  -H "Authorization: Bearer <your-access-token>"
```

### Добавление в избранное
```bash
curl -X POST http://127.0.0.1:7757/favorites \
  -H "Authorization: Bearer <your-access-token>" \
  -H "Content-Type: application/json" \
  -d '{"film_id": 1}'
```

### Удаление из избранного
```bash
curl -X DELETE http://127.0.0.1:7757/favorites \
  -H "Authorization: Bearer <your-access-token>" \
  -H "Content-Type: application/json" \
  -d '{"film_id": 1}'
```

### Получение всех фильмов
```bash
curl -X GET http://127.0.0.1:7757/films
```

### Получение топ-25 фильмов
```bash
curl -X GET http://127.0.0.1:7757/top25
```

### Получение фильма по ID
```bash
curl -X GET http://127.0.0.1:7757/films/1
```

### Получение всех актеров
```bash
curl -X GET http://127.0.0.1:7757/actors
```

### Получение актера по ID
```bash
curl -X GET http://127.0.0.1:7757/actors/1
```

### Обновление access токена
```bash
curl -X POST http://127.0.0.1:7757/refresh-token \
  -H "Content-Type: application/json" \
  -d '{"refreshToken": "<your-refresh-token>"}'
```

### Выход
```bash
curl -X POST http://127.0.0.1:7757/logout \
  -H "Content-Type: application/json" \
  -d '{"refreshToken": "<your-refresh-token>"}'
```

## ❌ Ошибки

API возвращает стандартные HTTP статусы:

| Статус | Описание |
|--------|----------|
| 200 | Успешный запрос |
| 201 | Успешное создание ресурса |
| 204 | Успешное удаление (нет содержимого) |
| 400 | Ошибка валидации данных |
| 401 | Неавторизован (токен не предоставлен или недействителен) |
| 403 | Доступ запрещен (недействительный токен) |
| 404 | Ресурс не найден |
| 500 | Внутренняя ошибка сервера |

### Формат ошибок

```json
{
  "message": "Описание ошибки"
}
```

Или для валидации:

```json
{
  "errors": [
    {
      "msg": "Введите корректный email",
      "param": "email",
      "location": "body"
    }
  ]
}
```
